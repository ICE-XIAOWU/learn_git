# git基本操作

1. ##### git inti初始化仓库

   使用git init可以对仓库初始化，并且初始化成功之后，会生成一个.git的目录，该目录中存储这管理当前目录内容所需的仓库数据。在git中，一般称这个目录为“附属于该仓库的工作树”。

2. ##### git status 查看仓库状态

   git status 命令用于显示git仓库的状态。

3. ##### git add 向暂存区中添加文件

   git add命令将文件加入到暂存区(Stage或者Index)中，如果不使用git add命令将文件加入到暂存区，那么git仓库是无法管理该对象的。暂存区是提交之前的一个临时区域。

4. ##### git commit 保存仓库的历史记录

   git commit命令可以将当前暂存区中的文件实际保存到仓库的历史记录中，可以通过这些记录，在工作树中复原文件。

   - 记录一行提交信息，在git commit后面加个 -m 参数。
   - 记录详细提交信息，在git commit后面直接回车

5. ##### git log 查看提交日记

   git log命令可以查看以往仓库中提交的日记。包括可以查看什么人在什么时候进行了提交或者合并，以及操作前后的差别。

   - 只显示提交信息的第一行： git log --pretty=short
   - 显示指定目录、文件： git log 目录名
   - 显示文件的改动： git log -p，加个目录名可以查看该文件提交前后的差别

6. ##### git diff 查看更改前后的差别

   git diff可以查看工作树、暂存区、最新提交之间的差别。+ 号是新添加的行，- 号是被删除的行。

   - 查看工作树和暂存区的差别： git diff
   - 查看工作树和最新提交的区别： git diff HEAD，查看本次提交和上次提交的差别，HEAD是指向当前分支中最新一次提交的指针。



# 分支的操作

1. ##### 分支是啥？

   - 分支是git中默认带有的，master是git默认创建的一个分支，所有的开发都是基于这个分支为中心进行的

   - 不同的分支中，可以进行不同的开发，每个分支都存在着自身最新的代码，等该分支完成开发后再与master分支合并。

   - 分支创建图如下：

   - 分支完成开发后合并到master分支的状态图：

     ![image](https://raw.githubusercontent.com/ICE-XIAOWU/picxImg/master/picxImg/image.4zut8kawsfw0.webp)

2. ##### git branch 显示分支一览表

   git branch可以将分支名以列表的方式显示，同时确认当前的所在的分支。在左边有带*号的就是当前的分支。

3. ##### git checkoout -b 创建、切换分支

   - Git checkout -b 分支名 ：创建分支并切换到新创建的分支。
   - git branch 分支名 +  git checkout 分支名： 也能取得一样的效果。
   - 培育分支：对master以外的一个分支不断的进行开发、修改、提交的操作就称之为培育分支。
   - git checkout 分支名： 切换分支
   - git checkout -/分支名： 可以切回到上一个分支

4. ##### 特性分支

   特性分支是集中实现单一特性，除此之外不进行任何作业的分支。如下图：

   ​	![image](https://raw.githubusercontent.com/ICE-XIAOWU/picxImg/master/picxImg/image.48x1g2xeqjq0.webp)

5. ##### 主干分支

   主干分支即是特性分支的原点，也是合并分支的终点。一般master为主干分支。

6. ##### git merge 合并分支

   首先切换回master分支，然后在使用git merge --no-ff 分支名 进行分支的合并，--no-ff是为了在历史记录中记录下本次分支的合并。

7. ##### git log --graph 以图表形式查看提交日志

   git log -- graph可以以图表的形式查看提交日志

8. ##### git reset 回溯历史版本

   使用git reset --hard 目标时间点的哈希值 的命令，即可回溯到该时间点的状态。**注意：哈希值只用输入4位以上就可以。**

9. ##### git reflog 查看当前仓库的操作日志

   - 与git log的区别：git log只能查看当前状态为终点的历史日记，git reflog可以查看当前仓库执行过的操作日志。
   - 只要不进行git的GC，就可以随时通过git reset命令回溯到指定目标的时间点

10. ##### 解决合并冲突的问题

    在实际开发中，往往需要删其中之一的冲突内容，所以在处理冲突时，一定要仔细的进行修改。

11. ##### git commit --amend 修改提交信息

    git commit --amend命令之后会启动一个编译器，这是就可以修改上次提交信息的备注信息。

12. ##### git rebase -i 压缩历史

    - 使用git commit -am "注释信息" ：可以一次性完成提交到暂存区以及保存到仓库历史的操作。
    - 使用git rebase -i HEAD~2命令可以选取当前分支中最新提交在内的两个历史记录为对象，并且在编译器中打开，可以直接修改需要编辑历史记录。保存成功后会显示rebase..
